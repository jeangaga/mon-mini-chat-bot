ARC ONE-PAGE MACRO SHEET PROMPT

You are producing a ONE-PAGE MACRO SHEET for a single macro unit
(country, region, or aggregate block such as US, Eurozone, Germany, Japan, China, Canada, Brazil, Poland, etc.).

The output must combine two time horizons:

Rolling Macro Dashboard covering the last 2–3 weeks
YTD Frozen Arc covering the year-to-date period

The goal is to create a page that is both:

decision-useful now
easy to memorize over time

Do not rewrite the full archive.
Do not produce a long macro note.
Compress the material into a stable one-page object.

This prompt must aim for the same level of compression, clarity, and structure as the EUR one-page sheet quality example, which is the benchmark format.

INPUT

You may receive:

weekly monitoring files
live macro files
release blocks
prior dashboard summary
prior frozen arc summary
weekly delta updates

Use only the material provided unless I explicitly ask for web verification.

CORE OBJECTIVE

Produce a one-page sheet that answers:

What is the current regime?
What changed over the last 2–3 weeks?
What is the key contradiction in the current macro mix?
What are the 3 most important recent releases?
What is the next gating condition?
What is the YTD macro arc in a few regime phases?
HARD RULES
no invented facts

do not include release counts
create a new YTD arc paragraph only when there is a genuine regime change
if the current regime is a continuation, extend the prior arc paragraph rather than inventing a new one
keep the whole output one-page / one-screen in spirit

YTD ARC CONSTRUCTION RULE

The YTD FROZEN ARC is not a chronology of every week.
It is a compressed sequence of regime phases.

Each paragraph must contain:

the data window
the dominant regime description
what changed materially in that phase

Do not create a new paragraph unless one of these changed materially:

growth regime
inflation regime
policy regime
labor regime
market/pricing interpretation regime

If the period is a continuation, extend the prior phase rather than creating an artificial new one.
